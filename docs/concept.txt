하나의 프로젝트를 진행 하려고 하는데, 처음이라 가능 한지 확인을 하고 싶어.
아래에 내가 하고 싶은 방향성을 적어 줄테니, 검토 해줘.
모든 구성 및 진행은 클로드코드를 활용하여 도움을 받아서 방향성을 잡고 절차를 진행 할 예정이고, 일단 컨셉 부터 잡아보자.
=================================
1. 현재 시스템 절차
- 발전소에서 GT, ST가 낼 수 있는 최대 출력(generator MW)값으로 입찰을 하는 시스템으로 구성되어있음.
- 그럼 이 최대 출력은 어떻게 산출하냐?
- 발전소 자체 적으로 공급가능용량 테스트를 진행하여 입찰을 위한 데이터를 산출함.
- 우리 발전소의 경우는 공급가능용량 테스트를 1주에 한번 또는 2주에 한번 이런식으로 주기 적으로 시행함.
- 공급가능용량 테스트를 시행한 후 테스트 결과를 산출 하는 엑셀 파일(엑셀1 이라고 지칭하겠음)이 있음
- 이 엑셀1 파일에 테스트 날짜와 시간을 입력하면 우리가 갖고 있는 RIMS라는 시스템에서 자동으로 테스트 결과 값을 가져와서 정리를 해줌.
- RIMS라는 시스템은 발전소 전체에서 중요한 값들을 선정하여 remote에서 따로 관리 할 수 있도록 하는 시스템임(remote inspection monitoring system)
- DCS라는 제어 시스템이 있지만, DCS에서 RIMS로 값을 통신으로 보내주기 때문에 값에 대한 신뢰도는 어느정도 있음.
- 엑셀1 파일에서 RIMS에서 가져오는 데이터는 GT Compressor inlet temperature/pressure, 대기압, 상대습도, GT load, ST load, CC load, aux power, EBH, 진공, 외기온도, 암모니아슬립, 암모니아투입량, nox 값, GT exhaust pressure, C/T basin Temperature 등
- RIMS에서 가져온 데이터들을 활용하고 siemens 기종의 combined cycle power output and efficiency correction curves VS ambient temperature. 곡선 및 다양한 수식을 활용하여 base load test 실적 값을 만들어줌.
- 엑셀1 파일의 공급가능용량테스트(base load test) 결과에서 도출한 CC(combine cycle) 출력값과 엑셀2 파일에서 산출한 CC 출력값을 비교하고 있음
- 그럼 엑셀2 파일에서 CC출력값을 어떻게 산출하냐?
- 엑셀2 파일 또한 특성 곡선을 활용하여 출력값을 계산하는걸로 추정함.(엑셀2는 대기압만 넣으면 자동으로 온도별 공급 가능 용량이 산출 된다.)
- combined cycle power output and efficiency correction curves VS ambient temperature. 곡선과 actual operating hours(AOH) of combined cycle 곡선등을 활용하는 것 같음
- 데이터는 온도별(-20도 부터 40도)로 1도 단위로 출력값이 산출 된다.(총 60개의 온도별)
- 엑셀2에서 출력값이 산출 되면, 엑셀1 파일에서 계산한 출력값과 동일 온도에서 비교를 한다.
- 이때 두개의 출력이 만약 2MW 차이가 난다면(실제 테스트 결과가 2MW가 더 적다면), 차이가 나는 2MW를 엑셀2에서 계산된 각 온도별 출력값에서 일정하게 -2MW를 한다.(-20도 부터 40도 범위의 각 온도마다의 출력값 60개에서 -2MW를 한다.)
- 차이를 반영한 60개의 온도별GT, ST output값을 엑셀3에 복사해서 붙여 넣기 하면 온도 profile 결과가 나온다.
- 엑셀3에서 결과는 발전소 Mode 별로 산출이 자동으로 된다.
- 이때 엑셀3에는 외부 weather 사이트에서 크롤링하여 가져온 일주일간의 시간대별(3시간 주기로 8개 time) 대기압도 기입한다.(대기압 크롤링하여 엑셀로 완성하는 프로그램은 파이썬 코드로 개발 완료)
- 결국 엑셀3의 데이터를 최종 입찰에 사용한다.
2. 문제 인식
- 발전소 특성상 낼 수 있는 출력보다 더 입찰을 하거나, 덜 입찰 하게 되면 손해 및 패널티가 있을 수 있다.
- 내가 알기로는 turbine의 경우 온도, 대기압 등의 영향으로 각 온도별 출력이 일정하지 않을 수 있다.
- 현 시스템 처럼 위에서 말한 60개의 온도별로 출력 차이(ex. 2MW)를 일괄적으로 반영을 하면 출력 정확도가 떨어 질 수 있다고 판단한다.
- 따라서 테스트 결과와 특성곡선으로 계산한 출력 차이를 일괄적으로 반영하는게 아닌, 공급가능용량테스트 데이터를 축적하여 각 온도별로 편차를 보정하고 반영 할 수 있는 시스템을 구축하고자 한다.
- 또한, 현재는 대기압 크롤링 엑셀파일 1개, 위에서 말한 엑셀1, 엑셀2, 엑셀3 파일등 모든 과정이 4개의 엑셀 파일로 모두 분산화 되어 있어 일 처리가 단조롭지 않다.

3. 목적
- 여러개의 파일로 나누어진 현 시스템을 간단하게 하나의 툴로 만들어서 업무를 단순화 하고자 함.
- 공급가능용량테스트가 끝난 후 날짜 시간만 입력하면 위에서 말한 엑셀1, 엑셀2, 엑셀3 과정을 모두 자동으로 처리해서 최종 온도 profile 파일을 생성하면 업무가 간단 해 질 것 같음.
- 대기압 크롤링 파일은 회사 외부망에서 돌아가기 때문에, 외부망에서 실행하고 생성된 엑셀 파일을 툴에 업로드 하는 방식으로 구성 하는 것도 생각하고 있음.
- 엑셀1 파일과 같이 위의 자동 처리 과정에서 RIMS에서 여러 데이터를 가져 오는 것도 단순화 및 자동화 하고 싶음.
- 공급가능용량테스트 데이터를 축적하여 각 온도별 CC 출력값을 보정하여 실제 발전기가 낼 수 있는 출력과 근접한 값을 도출 하고 싶음.
- 이론값은 온도/대기압 등 환경적인 요인으로 산출 되고, 실제 운전시에는 현장 제반 여건에 따른 변수로 이론값과 실제출력 간의 편차가 발생하므로 이 편차를 줄이고자함.
- 현재 실제 측정값과 이론값의 차이를 온도별로 집계하여 보정하는 엑셀을 따로 만들고 있음
- 누적된 공급가능용량테스트 결과 건수는 32건 정도이고, 테스트를 진행 할 수록 건수는 증가 할 예정임.
- 해당 보정 엑셀은 겨울:실제>이론(보정값 양수+), 여름:실제<이론(보정값 음수-), 복수기 진동도 차이 반영 등으로 보정값 산출
- CC 순출력= CC_ISO(설계 최대 출력, 450.4MW) / T_CoRR(온도보정) / P_CORR(대기압보정) / Deg(열화, 1.028) /RH_CORR / CP_CORR + W(IGV turn up) +보정값(실측 32건 기반 현실 조정, 겨울+, 여름-) - 소내전력(10MW)
- 결론은 일정한 툴(프로그램, html 등 UI가 있는)에서 대기압 크롤링 엑셀 파일을 업로드하고 날짜, 시간을 입력한 뒤 실행을 하면 최종적인 온도 Profile을 생성 해 준다.

4. 추가 사항
- 위에서 언급한 모든 엑셀파일을 클로드코드에 업로드 하여 검토를 진행 할 예정(엑셀에 포함된 모든 수식과 데이터를 검토 해야 프로젝트 진행이 가능 할 것 같음)
- 내가 위에서 적은 내용과 업로드하여 검토한 내용을 바탕으로 프로젝트의 방향성과 컨셉을 설정 할 수 있을거라고 생각함.
- 엑셀 파일은 추후 업로드 할 예정임.
