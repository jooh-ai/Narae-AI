## 3. 결정 필요 / 시운전 항목 (미해결)

### A. ✅ 이론기준값 기준 불일치 (M1) — **해결됨 (2026-06-26)**
- **원인(재규명)**: 누락 항은 복수기(CP)가 아니라 **상대습도(RH) 보정**이었음. base_cc에 복수기 설계항은
  이미 포함, 테스트별 cp_design ≈ 설계곡선(비율≈1). 시드 이론기준값(I)은 **실측 RH로 계산**된 것이었는데
  엔진은 RH 미반영이라 어긋남(겨울 저습→과대, 여름→과소, 우연한 온도계통오차로 보임).
- **조치**: `theory.theory_cc(..., rh=)`에 습도보정 `K_rh(rh,cit)` 추가.
  `store.record_test`가 **테스트 실측 RH**로 이론기준값 계산 → 시드 수기값과 정합.
- **검증**: 이론기준값 재현 오차 **2.63 → 0.19 MW**, 구간 보정값 수기 vs 엔진 차이 **최대 0.09 MW**(±0.5 내).
  → 시드(수기)·신규(엔진+RH) 동일 기준, DB 일관성 확보. **재기준화 불필요.**
- **Profile 불변**: 입찰 Profile은 RH 미지정(=RH 60) → base_cc 그대로, 엑셀4 검증값 유지.
- **참고**: W(IGV)·AB6(−2) 가산항은 이론기준값에 포함하지 않음(시드 I에 없음, 추가 시 오차 악화 확인).

### B. RH_CORR / 복수기진동 (concept §44) — 시운전 항목
- 현재 RH 60% 고정, 복수기 효과는 보정값에 흡수(설계 CP 기준 시). A의 CP 보정 구현 시 함께 정리.

### C. 사내 결선 검증 (DEPLOY §3·§5) — 사내 1순위
- excel_addin `CELL_MAP`(AG9·H8·J8·O8…)을 실제 엑셀1과 대조, 수동 취득값과 일치 확인.
- fnTagStat 비동기 취득이 `calculate()` 후 즉시 읽기에서 stale 일 수 있음 → 검증(숫자 가드)로 1차 방어, 필요시 폴링 추가.
- 패키징(.exe)은 xlwings/pywin32 hidden-import 추가 + 실제 exe로 스모크 테스트.
